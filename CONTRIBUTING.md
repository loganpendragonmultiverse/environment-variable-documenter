# Contributing

Use pull requests for focused syntax, reporting, security, or documentation improvements. Detection changes need realistic positive and false-positive tests. Any change touching dotenv handling must prove that real value-bearing files remain unread and template values remain absent from every report format.

Run the complete command set in `DEVELOPMENT.md`. Do not include real secrets, proprietary source, private `.env` files, or customer configuration names in public fixtures.

The maintainer reviews correctness, privacy, security, scope, documentation, tests, and maintainability before merging.
